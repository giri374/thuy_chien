Hiện tại, tôi sử dụng hệ thống Cell và Grid khá phụ thuộc UI, không tập trung vào 1 code logic chung.

Hãy tạo các thành phần Core chứa các class không kế thừa MonoBehaviour, chỉ dùng struct + class thông thường, lưu trữ dữ liệu của Cell, Grid và Ship. Làm sao cho các dữ liệu này tách biệt với UI và có thể gửi đi trong chế độ multiplayer online. Đồng thời viết thêm mapping View ↔ Core.

GridManager (chỉ là View): tạo CellData → render sprite/animation.
BattleSceneLogic thay vì dùng GridManager trực tiếp → dùng 2 Board:
"private Board player1Board = new Board();
private Board player2Board = new Board();"
