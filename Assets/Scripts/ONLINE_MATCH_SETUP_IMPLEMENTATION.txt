MENU → SETUP → ONLINE CONNECTION → BATTLE
─────────────────────────────────────────

Step 1: MenuScene
  └─ Người chơi chọn "Play Online" button
  └─ GameManager.SetGameMode(GameMode.PlayOnline)
  └─ Chọn Normal/Advanced Map
  └─ → SetupSceneUIManager.GoToBattle() → Start Setup Flow

Step 2: WeaponSetupScene (nếu AdvancedMap) + SetupScene
  └─ Cả Host và Guest đều qua các setup scene này (cách PlayWithBot/PlayWithFriend)
  └─ Người chơi chọn vũ khí, đặt tàu
  └─ Lưu placements vào GameManager.GetPlacements(1) [local player 1 data]
  └─ Lưu selected weapons vào GameManager.GetSelectedWeapons(1)
  └─ Confirm setup → SetupSceneLogic.GetNextAction() → GameFlowAction.GoToOnlineConnection

Step 3: OnlineConnectionScene via Netcode Scene Manager
  └─ Host khoảng host connection:
     ├─ MatchHostConnectionMenuViewController: Create Match
     ├─ → MatchHostConnectionMenu.CreateMatch_Async() → EConnection.StartConnection_Async()
     ├─ → HostConnection.StartConnection_Async() → NetworkManager.StartHost()
     ├─ → Join Code được hiển thị
     └─ UI Status: "Waiting for opponent..." → "Opponent connected. Syncing setup..."
  
  └─ Guest join connection:
     ├─ MatchGuestConnectionMenuViewController: Nhập Join Code
     ├─ → MatchGuestConnectionMenu.ConnectToMatch_Async(joinCode)
     ├─ → EConnection.StartConnection_Async(joinCode) → NetworkManager.StartClient()
     └─ UI Status: "Connecting..." → "Connected. Waiting for host..." → "Setup syncing..."

Step 4: Data Sync (Netcode RPC + Spawn)
  └─ Host đã spawn EGameMatch prefab thông qua MatchSpawner (OnServerStarted)
  └─ MatchSpawner.OnServerStarted() → Instantiate(eGameMatchPrefab) → NetworkObject.Spawn()
  
  └─ Client (Host + Guest):
     ├─ EGameMatch.OnNetworkSpawn():
     ├─ → SubmitLocalSetupToHost()
     ├─ → Mỗi client gửi placements + weapons
     
  └─ Host (Server):
     ├─ Host submit own setup trực tiếp: _setupByClientId[0] = hostSetup
     ├─ Guest submit via RPC: SubmitLocalSetup_ServerRpc() →  _setupByClientId[1] = guestSetup
     ├─ Server kiểm tra AllPlayersAreStillConnected() + cả 2 setup có đầy đủ
     ├─ → SyncSetup_ClientsAndHostRpc(player1Setup, player2Setup)
     
  └─ Client (Both):
     ├─ Nhận SyncSetup RPC
     ├─ → ApplySyncedSetup(): Populate GameManager với official data
     ├─ → IsSetupSynced = true
     ├─ → OnSetupSynced?.Invoke()
     └─ UI Status: "Setup synced. Starting battle..."

Step 5: Load BattleScene via Netcode Scene Manager
  └─ Host: LoadBattleSceneOnce() → NetworkManager.SceneManager.LoadScene(BattleScene)
  └─ Guest: Nhận scene load command, tự động load cùng scene
  └─ Dữ liệu đã sync trong GameManager:
     ├─ Player1Placements (Host tàu)
     ├─ Player2Placements (Guest tàu)
     ├─ Player1SelectedWeapons (Host vũ khí)
     └─ Player2SelectedWeapons (Guest vũ khí)

Step 6: BattleScene Setup
  └─ BattleSceneLogic.LoadAllShips():
     ├─ LoadShipsFromData(GameManager.GetPlacements(1), player1Grid)
     ├─ LoadShipsFromData(GameManager.GetPlacements(2), player2Grid)
     └─ Tàu được đặt đúng vị trí từ sync data
  
  └─ BattleWeaponManager setup:
     └─ Load vũ khí từ GameManager.GetSelectedWeapons(playerIndex)

3. KEY DECISIONS & FIXES
═══════════════════════════════════════════════════════════════════════════════

1. Scene Loading
   Decision: Use NetworkManager.SceneManager.LoadScene() cho online mode
   → Đảm bảo NetworkObjects spawn đúng + scene migrations sync
   Fallback: Standard SceneManager.LoadScene() cho single-player

2. Host Setup Submission
   Decision: Host submit trực tiếp vào _setupByClientId, Guest via RPC
   → Tránh race condition: Host + Client RPC không always sync ngay
   Result: AllPlayerClientIds[1] auto-set khi Server nhận RPC từ guest

3. Singleton Cleanup
   Decision: RuntimeInitializeOnLoadMethod reset + OnDestroy cleanup
   → Tránh stale singleton lúc multi-play mode (chạy chung 1 máy)
   Result: Mỗi lần Start Play, singleton sạch sẽ, không conflict

4. Connection Blocking
   Decision: WaitForOpponentRoutine + ConnectionWatchRoutine polling
   → Keep UI responsive, block khi disconnect
   Result: Status updates + Retry/Back button điều khiển
   Note: Not using async Task - dùng Coroutine để dễ cancel

6. ARCHITECTURE NOTES
═══════════════════════════════════════════════════════════════════════════════

Host = Server + Client (IsServer=True, IsClient=True, IsHost=True)
Guest = Client only (IsServer=False, IsClient=True, IsHost=False)

Mapping:
  Player 1 = Host (always) → clientId 0
  Player 2 = Guest → clientId 1 (dynamic, set from OnClientConnectedCallback or RPC)

RPC Types Used:
  [Rpc(SendTo.Server)]:      Client sends to server (SubmitLocalSetup_ServerRpc)
  [Rpc(SendTo.ClientsAndHost)]: Server broadcasts to all (SyncSetup_ClientsAndHostRpc)

Command Pattern (for next phase):
  ✓ AttackCommand: Immutable attack data (position, weapon, attacker)
  ✓ AttackCommandHistory: Log for replay/sync
  ✓ AttackCommandExecutor: Execute & callback result
  → Will use NetworkList<CommandData> hoặc RPC per command
